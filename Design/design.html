SBL: Symbolic Basic Language

SBL is a language that extends racket, providing the ability to define functions that can take both concrete and symbolic values. The programmer can interact with these functions either by using the outputs that SBL provides or by implementing callback function.

1 Design

1.1 SBL

1.1.1 Vocab

  • define-symbolic: for defining symbolic functions and values

  • variable: a name that is defined

  • get-input: primitive function that gets path between functions

  • symb-expr: an expr that triggers a symbolic evaluation

  • define-callback: adds a callback on application of symbolic funtion

  • racket-expr: racket-expr

  • bsl–expr: an expression that only uses the grammar vocab and semantics of BSL-

1.1.2 Grammar

  • (define-symbolic symb-expr)

  • (define-symbolic (variable ...) symb-expr)

  • (get-path variable variable symb-expr)

  • (define-callback variable)

  • (get-input variable variable symb-expr)

  • (variable (symb-expr U racket-expr) ...)

1.1.3 Semantics

  • define-callback: adds a callback for each application of a symbolic function

  • get-input: produces a concrete value for all symbolic values such that applying the first given function to these inputs reaches the second function. The symb-expr must be the application of the first variable.

  • (variable expr ...): applies the function to the given expressions if the variable is an application of a symbolic function all expressions must evaluate to either a symbol or number
