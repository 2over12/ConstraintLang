SBL: Symbolic Basic Language

Ian Smith and Eddy Li

 #lang SBL

Added in version 1.0.
Changed in version 1.1: Added examples

1 Introduction

SBL (pronounced Sibyl) is a language that extends Racket, providing the ability to define functions that can take both concrete and symbolic values on the fly. The programmer can interact with these functions by using the outputs that SBL provides or by interacting with out primitive functions.

2 SBL

2.1 SBL Vocabulary

  Vocab = define-symbolic
  | var
  | get-input
  | symb-expr
  | bsl-expr

  • A define-symbolic is for defining symbolic functions.

  • A var is a name that can be bound.

  • A get-input is a primitive function that gets the requisite inputs to reach a given function from the given application.

  • A symb-expr is an expression that evaluates to either concrete number or a symbolic value.

  • A bsl-expr is an expression written in in BSL-. For more information, see BSL-.

2.2 SBL Grammar

  Functions = (define-symbolic (variable ...) bsl–expr)
  | (get-constraint variable symb-expr)
  | (get-inputs variable symb-expr)
  | (variable (symb-expr) ...)

All of the following syntactic forms assume the following functions have been defined in their examples
(define-symbolic (g a) 42)
(define-symbolic (t b) (if0 (> b 4) 4 2))
(define-symbolic (c a) (+ a 7))
(define-symbolic (x c d) (if0 (>= (c d) 4) (t c) (g 1)))
(define-symbolic (f a b) (if0 (< (sub1 a) 10) (x a b) 4))

syntax

(define-symbolic (id var ...) bsl-expr)

Defines a function that takes in the vars you pass in and executes the BSL-expr, replacing any mention of the vars in the bsl-expr with the values you passed in.

syntax

(get-constraint var symb-expr)

Produces the constraints on which you can reach a function through the symb-expr passed in. If you can’t reach the var from the symb-expr passed in, then you get #f.

For Example:
> (get-constraints g (g b))
#t
 
> (get-constraints (f 100 e))
#f
 
> (get-constraints (f i e))
(&& (< (+ 1 i) 10) (! (<= 4 (+ 7 e))))

syntax

(get-inputs var symb-expr)

Produces a set of values that can reach the given symbolic application. If any value will do, then return (model) by itself

For Example:
> (get-inputs g (g b))
(model)
 
> (get-inputs g (f 100 e))
(unsat)
 
> (get-inputs g (f i e))
(model
 [i 8]
 [e -4])

syntax

(variable symb-expr ...)

Applies the function to the given expressions if the variable is an application of a symbolic function. All expressions must evaluate to either a symbol or number. The produced result is the application of the function on the constraints, fully expanded.

For Example:

> (f i e)
 
(ite (< (+ 1 i) 10) (ite (<= 4 (+ 7 e)) (ite (< 4 i) 4 2) 42) 4)
 
>(f 1 2)
 
2
 
>(f i 2)
 
(ite (< (+ 1 i) 10) (ite (< 4 i) 4 2) 4)

2.3 SBL Scope

  • (define-symbolic variable expr) binds variable for all below top level forms that are symbolic.

  • (define-symbolic (variable variable ...) bsl-expr) binds the first variable in all top level forms and the rest of the variables in bsl-expr.

A note on recursion: In the current build of SBL, recursion of any form is not allowed. Functions can only be called below where they are defined, and cannot refer to themselves.

3 BSL-

BSL- is a version of the Beginner Student Language that only support numbers and symbolic expressions, and does not support recursion.

4 Examples

4.1 Rosette Fun

Here is a really cool example that illustrates how Rosette changes how the language evaluates things.

(define-symbolic (func1 c d) (if0 (<= c d) 12 3))
(define-symbolic (func2 a) (func1 (func1 3 4) a))
(func2 (func1 x 2))

Evaluates to

(ite (<= x 2) 12 3)

While
(define-symbolic (func1 c d) (if0 (< c d) 12 3))
(define-symbolic (func2 a) (func1 (func1 3 4) a))
(func2 (func1 x 2))

Evaluates to

3

Why?

Well, let’s walk through what happens.

In the first example, func2 is given (func1 x 2) and so evaluates to

(func1 (func1 3 4) (func1 x 2))

Then, (func1 3 4) and (func1 x 2) are evaluated, which results in

(func1 12 (if0 (<= x 2) 12 3))

Which then evaluates to

(if0 (<= 12 (if0 (<= x 2) 12 3)) 12 3)

Rosette already knows that the only results that the inner if0 can produce are 12 and 3. Since 12 could be less than or equal to the result of (if0 (<= x 2)), Rosette doesn’t pre-emptively know what the result will be, and therefore returns the expanded Rosette version of if0, which on the inside is an if-then-else in the form of (ite (<= f 2) 12 3), since it has already figured out that the result of the outside ite is dependant on, and is essence the same result as what would happen with just (<= x 2), and thus just returns

(ite (<= x 2) 12 3)

Meanwhile, in the second example, the exact expansion happens, except with <= replaced with <. This doesn’t chnage anything until

(func1 12 (if0 (< x 2) 12 3))

Which then evaluates to

(if0 (< 12 (if0 (< x 2) 12 3)) 12 3)

Rosette already knows that the only results that the inner if0 can produce are 12 and 3. Since neither 12 nor 3 are strictly < than 12, Rosette knows that no matter the result of the inner if0, the result will be 3, and therefore returns 3.
